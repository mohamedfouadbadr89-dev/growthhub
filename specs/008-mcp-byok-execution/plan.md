# Implementation Plan: MCP Execution Layer with BYOK Support

**Branch**: `008-mcp-byok-execution` | **Date**: 2026-04-27 | **Spec**: [spec.md](./spec.md)

## Summary

Add an MCP (Model Context Protocol) execution layer that allows orgs to bring their own AI provider keys (BYOK). New backend routes handle encrypted key storage via Supabase Vault and route prompts ‚Äî enriched with org-specific tool context ‚Äî to OpenAI, Anthropic, or OpenRouter. A BYOK modal in the Integrations UI provides the key management UX. All additions are isolated new files; two minimal line additions are required to register the new route in the backend index.

## Technical Context

**Language/Version**: TypeScript 5.x (Node 20)
**Primary Dependencies**: Hono 4.7, openai SDK 4.104 (already installed ‚Äî covers all 3 providers), @supabase/supabase-js 2.49
**Storage**: Supabase Vault for key encryption; `organizations` table for vault ID reference; in-process Map for cache + rate limiter
**Testing**: Manual integration testing per quickstart.md scenarios
**Target Platform**: Railway (backend), Hostinger VPS (frontend)
**Project Type**: Web service ‚Äî additive backend routes + React component
**Performance Goals**: Cache hits < 200ms; non-cached execution < 30s (provider-bound)
**Constraints**: Zero new npm dependencies; no schema changes beyond additive migration; no modifications to existing files except two-line route registration
**Scale/Scope**: Per-org rate limit 60 req/min; cache TTL 5 minutes

## Implementation Phases

### Phase 1: Backend Infrastructure (no UI)

**Goal**: `/api/v1/ai/execute` and `/api/v1/ai/connect` work end-to-end.

1. **`supabase/migrations/20260427000008_mcp_byok.sql`**  
   Add `vault_byok_mcp_provider` and `vault_byok_mcp_secret_id` columns to `organizations`.

2. **`backend/src/lib/ai-cache.ts`**  
   `get(key)`, `set(key, value, ttlMs)`, `has(key)` ‚Äî Map-based, auto-evicts on read.  
   Cache key: `${orgId}:${sha256hex(provider + model + prompt)}`

3. **`backend/src/lib/rate-limiter.ts`**  
   `check(orgId): boolean` ‚Äî returns `true` if under limit, increments counter.  
   `retryAfterMs(orgId): number` ‚Äî ms until window resets.  
   Sliding window: 60 req/60s per org (env-configurable).

4. **`backend/src/services/ai/ai-client.ts`**  
   `createAiClient(provider, apiKey)` ‚Üí OpenAI SDK instance with correct baseURL.  
   Providers:
   - `openai` ‚Üí `https://api.openai.com/v1`
   - `anthropic` ‚Üí `https://api.anthropic.com/v1` (OpenAI-compatible)
   - `openrouter` ‚Üí `https://openrouter.ai/api/v1` (same as existing `openrouter.ts`)

5. **`backend/src/services/ai/mcp-tools.ts`**  
   Export `MCP_TOOLS: ToolDefinition[]` array with 3 tools.  
   Export `callTool(name, params, orgId)` dispatcher.  
   Tool handlers query `supabaseAdmin` scoped to `orgId`.

6. **`backend/src/services/ai/prompt-builder.ts`**  
   `buildPrompt({ userPrompt, orgId, context? }): { system: string; userMessage: string }`  
   System section always includes:
   - "You MUST NOT autonomously execute any actions."
   - "You MUST NOT fabricate data. Only reference data returned by tools."
   - Org ID and optional page context

7. **`backend/src/routes/v1/ai.ts`**  
   - `GET /connect` ‚Üí status check (connected/provider)  
   - `POST /connect` ‚Üí store key in vault, update org row  
   - `DELETE /connect` ‚Üí delete vault secret, null org columns  
   - `POST /execute` ‚Üí rate-limit ‚Üí cache ‚Üí vault ‚Üí prompt ‚Üí AI ‚Üí tool resolution ‚Üí cache ‚Üí respond

8. **`backend/src/routes/v1/index.ts` (2-line addition)**  
   ```ts
   import { aiRouter } from './ai.js'
   v1.route('/ai', aiRouter)
   ```

### Phase 2: Frontend Components

**Goal**: Integrations page shows AI Assistant card + BYOK modal works.

9. **`components/ai/byok-modal.tsx`**  
   Props: `{ isOpen: boolean; onClose: () => void; onSaved: (provider: string) => void }`  
   Fields: provider dropdown (OpenAI / Anthropic / OpenRouter), API key input (type=password).  
   Submit: `POST /api/v1/ai/connect` via `apiClient`.  
   On success: calls `onSaved(provider)`, closes modal.  
   Disconnect button (if already connected): `DELETE /api/v1/ai/connect`.

10. **`app/integrations/connect/page.tsx` (additive update)**  
    - Import `ByokModal`  
    - Add `aiConnected` and `aiProvider` state  
    - Add `showByokModal` state  
    - Add entry to `INTEGRATIONS` array:
      ```ts
      {
        id: "mcp-ai",
        name: "AI Assistant (MCP)",
        desc: "Connect your own AI provider key for BYOK execution with full org context.",
        category: "Analytics",          // or new "AI" category
        connected: aiConnected,          // from state, loaded on mount via GET /api/v1/ai/connect
        entities: ["Campaigns", "Creatives", "Actions"],
        iconBg: "bg-violet-50",
        iconColor: "text-violet-600",
        icon: <Sparkles className="w-6 h-6" />,
      }
      ```
    - Override the Connect button handler for `mcp-ai` to open `ByokModal` instead of calling `handleConnect`.
