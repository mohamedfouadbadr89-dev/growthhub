# Implementation Plan: Intelligence Layer — AI Decision Engine

**Branch**: `claude/init-growthhub-PaRUm` | **Date**: 2026-04-20 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `specs/003-intelligence-layer/spec.md`

## Summary

Build an event-driven AI decision engine that analyzes synced `campaign_metrics` data to detect anomalies (ROAS drops, spend spikes, conversion drops) and scaling opportunities. For each detected signal, the engine generates a decision record with an AI-produced explanation (via OpenRouter), a heuristic confidence score, and a recommended action. Threshold-based alerts run in parallel without AI. All generation is gated by billing plan: `subscription` orgs consume 1 credit per AI explanation; `ltd` orgs use their BYOK key from Supabase Vault. Decision generation is triggered after each integration sync (event-driven via Inngest) and on demand via a manual refresh API. Four frontend pages are wired to real data: Decisions Overview, Decision Detail, Alerts Center, Opportunities.

## Phase 0: Research Summary

All technical decisions resolved — see `research.md`. Key decisions:

1. **OpenRouter**: `openai@^4` npm package with custom `baseURL: 'https://openrouter.ai/api/v1'`. Default model: `google/gemini-2.0-flash-001`.
2. **Anomaly detection**: PostgreSQL window function query (`AVG() OVER (ROWS BETWEEN 7 PRECEDING AND 1 PRECEDING)`). Thresholds: ROAS_DROP ≥30%, SPEND_SPIKE ≥3×, CONVERSION_DROP ≥40%, SCALING_OPPORTUNITY ROAS>3.5x for 5+ days.
3. **Credit deduction**: Atomic `UPDATE organizations SET credits_balance = credits_balance - 1 WHERE org_id = $1 AND credits_balance >= 1 AND plan_type = 'subscription' RETURNING credits_balance`. Zero rows = insufficient credits.
4. **Inngest chaining**: `syncIntegration` sends `intelligence/decisions.requested` on success. New `generateDecisions` function handles both sync-triggered and manual-refresh paths.
5. **Priority score**: `severity_weight × (confidence_score / 100)` where ROAS_DROP=90, CONVERSION_DROP=85, SPEND_SPIKE=70, SCALING_OPPORTUNITY=40.

### Billing Gate (src/services/intelligence/decision-generator.ts)

```typescript
async function deductCreditAndGetClient(orgId: string): 
  Promise<{ client: OpenAI; didDeduct: boolean } | null> {
  
  const org = await supabaseAdmin
    .from('organizations')
    .select('plan_type, vault_byok_openrouter_secret_id')
    .eq('org_id', orgId)
    .single()

  if (org.plan_type === 'ltd') {
    if (!org.vault_byok_openrouter_secret_id) return null  // no BYOK key
    const key = await readSecret(org.vault_byok_openrouter_secret_id)
    return { client: getOpenRouterClient(key), didDeduct: false }
  }

  // subscription: atomic deduct
  const { data } = await supabaseAdmin.rpc('deduct_credit', { p_org_id: orgId })
  if (!data) return null  // zero credits
  return { client: getOpenRouterClient(process.env.OPENROUTER_API_KEY!), didDeduct: true }
}
```

Note: `deduct_credit` is a Postgres function defined in the migration for atomicity.
